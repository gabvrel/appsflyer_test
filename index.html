Running on Local File (file://)

AppsFlyer SDK cannot set cookies on file:// paths. This prevents the generation of an Anonymous Device ID and causes 400 Bad Request errors.

Fix: Use a local server (e.g., VS Code Live Server) or drag this file to Netlify Drop.

Connection Blocked / Brave Shields

Brave Browser or Ad Blockers often modify headers causing CloudFront 400 Errors.

Fix: Click the Lion icon (Brave Shields) -> "Turn OFF Shields" for this site.
